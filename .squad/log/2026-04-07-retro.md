### Root Causes

1. **Shared workspace without branch isolation.** All agents operated on the same filesystem. Without strict `git worktree` discipline or enforced checkout verification, commits, stashes, and branches bled across agent boundaries.

2. **No branch-before-commit discipline.** Multiple incidents тАФ Mickey's commit on Pluto's branch, Devcontainer work on Chip's branch тАФ share the same root cause: agents did not verify current branch before writing and committing.

3. **Feature-branch PRs.** PR #25 was opened against a feature branch (`squad/1`) rather than `develop`. When that branch was deleted on merge, the PR was orphaned. All PRs must target `develop`.

4. **Ralph did not pull before distributing work.** Remote merges via GitHub API advanced `origin/develop` without advancing the local clone. Subsequent agents worked from stale HEAD.

5. **No branch name validation.** The `dev` branch was created and pushed before anyone caught the naming error. An automated check would have caught this in seconds.

6. **CI red flags accepted silently.** The PSScriptAnalyzer job has been failing since early merges. No one blocked on it because there was no policy requiring all jobs to be green. There is now.

7. **Stash lifecycle not managed.** `git stash` was used as a conflict escape hatch with no corresponding pop/cleanup step in Ralph's round teardown.

---

### What Should Change

1. **Branch-before-commit rule:** Every agent runs `git branch --show-current` and confirms the expected branch before staging any file. This is non-negotiable.

2. **All PRs target `develop`:** No PR may be opened against a feature branch. If context from another branch is needed, cherry-pick or rebase onto `develop` first.

3. **Ralph pulls after every remote merge:** After merging any PR via the GitHub API, Ralph issues a `git pull origin develop` directive to all active agents before the next round begins.

4. **Branch name validation before push:** Ralph's task templates validate branch name against the pattern `(develop|squad\/.+)` before any `git push`. Fail loudly on mismatch.

5. **CI policy тАФ all jobs must be green for merge:** If a job is failing for pre-existing reasons, the issue must be tracked and labeled before the PR is approved. No merging through red CI.

6. **End-of-round stash cleanup:** Ralph's teardown sequence includes `git stash list` and explicit pop or drop for any stash left by that round's agents.

7. **Lightweight PR checklist in CONTRIBUTING.md:** Three items: CI green (or pre-existing failures tracked), branch targets `develop`, change scoped to assigned issue.

8. **Design review before parallel config work:** When two or more issues touch shared files (e.g., `install.sh`), Mickey runs a brief interface-alignment ceremony before branches are opened.
