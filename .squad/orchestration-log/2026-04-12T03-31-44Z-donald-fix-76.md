## Summary

Donald diagnosed that `printf 'y\n' | timeout 60 gh copilot` silently fails in non-TTY contexts (Devcontainer `postCreateCommand`) because `gh copilot` calls `isatty(stdin)` and ignores piped input when stdin is not a terminal. Replaced the invocation with `printf 'y\n' | timeout 120 script -q /dev/null -c "gh copilot"` so `script(1)` provides a pseudo-TTY and the piped `y` is accepted.

## Changes

- **File:** `scripts/linux/tools/copilot-cli.sh`

- **Timeout:** Increased from 60 s to 120 s to accommodate slower network downloads inside containers.
